# Intradevor

## Настройка HTTP-клиента

Если в логах появляются предупреждения вида
`HTTP attempt 1 failed: SocketTimeoutError('Timeout on reading data from socket')`,
сервер отвечает слишком медленно. Теперь таймауты и число повторов можно менять
через переменные окружения (по умолчанию таймауты увеличены, чтобы дать ответу
больше времени):

- `HTTP_TIMEOUT_TOTAL` — общий таймаут запроса (секунды, по умолчанию 30).
- `HTTP_TIMEOUT_CONNECT` — таймаут установки TCP-подключения (секунды, по умолчанию 10).
- `HTTP_TIMEOUT_SOCK_CONNECT` — таймаут рукопожатия сокета (секунды, по умолчанию 10).
- `HTTP_TIMEOUT_SOCK_READ` — таймаут чтения ответа (секунды, по умолчанию 20).
- `HTTP_MAX_RETRIES` — количество повторных запросов при сетевых ошибках (по умолчанию 5).
- `HTTP_RETRY_BACKOFF` — стартовая задержка между повторами в секундах (удваивается после каждого повтора).

Пример запуска с увеличенным таймаутом чтения:

```bash
HTTP_TIMEOUT_SOCK_READ=40 python main.py
```
